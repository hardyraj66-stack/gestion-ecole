👩‍🏫 Module Professeurs

Gestion des enseignants et de leurs affectations aux classes/matières

Schéma Mongoose

Collection : professeurs

ChampTypeContraintes

prenomStringrequired

API REST

GET /professeurs

Retourne la liste de tous les professeurs (actifs et inactifs).

[
  {
    "id": "...",
    "nom": "Dupont",
    "prenom": "Marie",
    "genre": "F",
    "email": "m.dupont@ecole.mg",
    "telephone": "034 12 345 67",
    "statut": "actif"
  }
]

GET /professeurs/:id

Retourne un professeur par son ID. 404 si introuvable.

POST /professeurs

Crée un professeur. Émet l'événement Socket.IO professeur:event et déclenche un rebuild de la vue ReadProfesseur.

// Body
{
  "nom": "Dupont",
  "prenom": "Marie",
  "genre": "F",
  "email": "m.dupont@ecole.mg",
  "telephone": "034 12 345 67"
}

PATCH /professeurs/:id

Mise à jour partielle. Mêmes événements qu'à la création.

PATCH /professeurs/:id/desactiver

Passe le statut à 'inactif'. Le professeur reste dans la base mais n'est plus sélectionnable dans les affectations.

PATCH /professeurs/:id/activer

Repasse le statut à 'actif'.

Affectations professeur–matière–classe

Les affectations sont gérées via un module dédié TeacherAssignment (contexte frontend TeacherAssignmentContext). Une affectation lie un professeur, une matière et une classe. Elle permet :

  • D'afficher les matières enseignées dans la fiche professeur
  • De pré-remplir le champ professeur dans les créneaux de planning
  • De griser les matières déjà affectées dans la vue d'affectation
