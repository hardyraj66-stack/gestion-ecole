🔧 Backend — Architecture générale

NestJS 10, MongoDB, Mongoose, Socket.IO, Seeder

Démarrage

cd server
npm install
npm run dev          # Développement (ts-node)
npm run build        # Compilation TypeScript
npm start            # Production (node dist/main.js)

Le serveur écoute sur le port 3000 (configurable via PORT). La connexion MongoDB est sur mongodb://localhost:27017/gestion-ecole (configurable via MONGO_URI).

Structure modulaire NestJS

Chaque domaine métier suit le pattern Module → Controller → Service :

AppModule ├── EventsModule (Global) → EventsGateway (Socket.IO) ├── SeederModule → SeederService (OnModuleInit) ├── AnneesModule → AnneesController + AnneesService ├── ClassesModule → ClassesController + ClassesService ├── ElevesModule → ElevesController + ElevesService ├── MatieresModule → MatieresController + MatieresService ├── NotesModule → NotesController + NotesService ├── PlanningModule → PlanningController + PlanningService └── SallesModule → SallesController + SallesService

Pattern commun des services

Chaque service injecte son modèle Mongoose et expose :

@Injectable()
export class XxxService {
  constructor(@InjectModel(Xxx.name) private model: Model<Xxx>) {}

  findAll()              → model.find().exec()
  findById(id: string)   → model.findById(id).exec()
  create(data: any)      → new this.model(data).save()
  update(id, data)       → model.findByIdAndUpdate(id, data, { new: true })
  delete(id)             → model.findByIdAndDelete(id)
}

Pattern commun des contrôleurs

@Controller('xxx')
export class XxxController {
  constructor(
    private readonly service: XxxService,
    private readonly events: EventsGateway,  // injecté globalement
  ) {}

  @Get()     findAll()
  @Get(':id') findById(@Param('id') id)
  @Post()    create(@Body() body)    → events.emit('xxx:created', item)
  @Patch(':id') update(...)          → events.emit('xxx:updated', item)
  @Delete(':id') delete(...)         → events.emit('xxx:deleted', { id })
}

Schéma Mongoose — Transformation JSON

Tous les schémas utilisent la même transformation pour exposer id au lieu de _id :

@Schema({
  timestamps: true,
  toJSON: {
    virtuals: true,
    transform: (_, ret) => {
      ret.id = ret._id;
      delete ret._id;
      delete ret.__v;
      return ret;
    },
  },
})

Seeder automatique

SeederService implémente OnModuleInit. Au démarrage, s'il n'y a aucune salle en base, il insère :

  • 1 année scolaire active (2024-2025)
  • 12 salles (standard, labo, info, sport, arts)
  • 15 matières avec coefficients et couleurs
  • 15 classes (6ème → Terminale)
  • ~350 élèves (22-28 par classe)
  • ~2000+ notes (trimestre 1)
  • ~200+ créneaux de planning

Le seeder ne s'exécute que si la base est vide. Les lancements suivants conservent les données existantes.

WebSocket Gateway

@WebSocketGateway({
  cors: { origin: '*' },
  transports: ['websocket', 'polling'],
})
export class EventsGateway {
  @WebSocketServer() server: Server;

  emit(event: string, data: any) {
    this.server.emit(event, data);
  }
}

Le module EventsModule est marqué @Global(), ce qui rend EventsGateway injectable dans tous les contrôleurs sans import explicite du module.
