CQRS appliqué

Le backend sépare lecture et écriture.

ResponsabilitéContrôleurCollections
ReadReadControllerread_classes, read_eleves, read_matieres, read_notes, read_creneaux, read_salles
WriteClassesController, ElevesController, etc.classes, eleves, matieres, notes, creneaux, salles, anneescolaires

Principe des read models

Chaque collection read contient 1 document par entité, enrichi par dénormalisation :

  • read_classes : + nb_eleves, taux
  • read_eleves : + classe_nom, classe_niveau
  • read_notes : + eleve_nom, eleve_prenom, matiere_nom, matiere_code
  • read_creneaux : + classe_nom

Chaque document read possède un champ source_id qui pointe vers l'_id write d'origine.

Transformation JSON

Write schemas

const transform = (_: any, ret: Record<string, any>) => {
  ret.id = ret._id;
  delete ret._id;
  delete ret.__v;
  return ret;
};

Read schemas

const transform = (_: any, ret: Record<string, any>) => {
  ret.id = ret.source_id;   // important : le frontend garde l'id write
  delete ret._id;
  delete ret.__v;
  delete ret.source_id;
  return ret;
};

ViewBuilder

Le ViewBuilderService reconstruit les collections read.

Au démarrage

  • rebuildAll() reconstruit toutes les projections à partir des collections write

À chaque écriture

  • Les write controllers appellent explicitement viewBuilder.onXxxWrite()
  • Ex. onEleveWrite() → rebuild read_eleves + read_classes

Change Streams (bonus)

  • Si MongoDB tourne en replica set, les Change Streams écoutent les collections write
  • Si MongoDB est en standalone, le backend logge un warning et utilise uniquement la synchro explicite depuis les write controllers

Important : le message The $changeStream stage is only supported on replica sets n'est pas bloquant. Le système reste fonctionnel grâce au déclenchement explicite du ViewBuilder après chaque écriture.

Logs API

Le middleware ApiLoggerMiddleware logge :

  • méthode HTTP
  • endpoint
  • timestamp
  • durée
  • origin / referer
  • IP
  • user-agent tronqué
[API] [GET] /read/dashboard → 200 (12ms) | origin: http://localhost:5173 | ip: ::1
[API] [POST] /eleves → 201 (45ms) | origin: http://localhost:5173 | ip: ::1

Seeder

Le seeder peuple uniquement les collections write. Les collections read sont générées ensuite par le ViewBuilder.
