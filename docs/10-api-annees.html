Gestion des années scolaires : machine à états, transitions, snapshot, audit

Schéma AnneeScolaire

{
  label:      String  (requis, unique)  // "2024-2025"
  debut:      String  (requis)          // "2024-09-02"
  fin:        String  (requis)          // "2025-07-05"
  statut:     Enum    ['active','terminee','preparation']
  historique: [{
    action:   String,   // 'creation' | 'demarrage' | 'cloture' | 'creation_auto'
    date:     String,   // ISO timestamp
    details:  String    // description lisible
  }]
}

Machine à états

┌──────────────┐ │ PRÉPARATION │ └──────┬───────┘ │ POST /:id/demarrer │ (vérifie: aucune année active) ▼ ┌──────────────┐ │ ACTIVE │ ← une seule à la fois └──────┬───────┘ │ POST /:id/terminer │ (double confirmation frontend) ▼ ┌──────────────┐ │ TERMINÉE │ ← lecture seule └──────┬───────┘ │ (automatique) │ crée la nouvelle année ▼ ┌──────────────┐ │ PRÉPARATION │ ← nouvelle année └──────────────┘

Endpoints API

MéthodeRouteDescription
GET/anneesToutes les années (tri desc)
GET/annees/activeL'année active ou null
GET/annees/:idDétail d'une année
GET/annees/:id/snapshotDonnées complètes de l'année
POST/anneesCréer en préparation
POST/annees/:id/demarrerPréparation → Active
POST/annees/:id/terminerActive → Terminée (+ crée suivante)
PATCH/annees/:idModifier (interdit si terminée)
DELETE/annees/:idSupprimer (seulement préparation)

Endpoint Snapshot

GET /annees/:id/snapshot retourne :

{
  annee:    AnneeScolaire,
  classes:  Classe[]       // filtrées par anneeScolaireId (avec fallback sur annee_scolaire pour données non migrées)
  eleves:   Eleve[]        // filtrés par classe_id ∈ classes
  notes:    Note[]          // filtrées par eleve_id ∈ eleves
  creneaux: Creneau[]      // filtrés par classe_id ∈ classes
  matieres: Matiere[]      // toutes (partagées)
}

Règles métier

RègleVérification
Une seule année activedemarrer() vérifie qu'aucune autre n'est active
Lecture seule une fois terminéeupdate() et delete() refusent si statut === 'terminee'
Suppression interdite si activedelete() vérifie le statut
Création auto de la suivanteterminer() calcule le label N+1 et crée en préparation
TraçabilitéChaque transition ajoute une entrée dans historique[]

Double confirmation (frontend)

  • Terminer : 1ère popup (warning) « Vous êtes sur le point de clôturer… » → 2ème popup (danger) « Confirmez-vous la clôture définitive ? »
  • Démarrer : popup (info) « Assurez-vous que tout est configuré… »
