🎓 Module Niveaux

Niveaux scolaires configurables — ordre d'affichage et matières autorisées par niveau

Rôle

Le module Niveaux est une couche de configuration qui permet de :

  • Définir les niveaux scolaires de l'établissement (6ème, 5ème, Terminale, etc.) avec un ordre d'affichage
  • Restreindre les matières disponibles en planning pour chaque niveau
  • Alimenter dynamiquement les listes déroulantes de niveau (création de classe, planning)

Schéma MongoDB — Collection niveaux

{
  nom:         String   (requis, unique)   // "6ème"
  ordre:       Number   (requis, défaut 0) // position dans les listes
  description: String   (défaut "")        // "Collège — 1ère année"
  matiere_ids: [String] (défaut [])        // IDs des matières autorisées
  createdAt, updatedAt                     // timestamps auto
}

matiere_ids vide = toutes les matières autorisées. Ce comportement est intentionnel : un niveau sans restriction laisse toute liberté dans le planning.

Format de réponse /read/niveaux

[
  {
    id:          "6a0c...",      // null si niveau orphelin (non configuré)
    niveau:      "6ème",         // nom du niveau
    ordre:       0,
    description: "",
    matiere_ids: ["abc", "def"],
    count:       4               // nombre de classes ayant ce niveau
  },
  {
    id: null,                    // présent dans des classes mais pas encore configuré
    niveau: "CP",
    ordre: null,
    description: "",
    matiere_ids: [],
    count: 2
  }
]

Niveaux orphelins : si des classes référencent un niveau qui n'existe pas dans la collection niveaux, ce niveau apparaît quand même dans la liste avec id: null. Il peut être configuré (créé) via la modale d'édition dans la page Niveaux.

Règles métier

RègleDétail
Unicité du nomDeux niveaux ne peuvent pas avoir le même nom (insensible à la casse)
Ordre consécutifÀ chaque création, modification ou suppression, les ordres sont réassignés de manière consécutive (0, 1, 2…) sans trous
Réorganisation automatiqueSi un niveau est inséré à l'ordre n, tous les niveaux existants à partir de n sont décalés pour maintenir la consécutivité
Suppression protégéeUn niveau qui a des classes associées (count > 0) ne peut pas être supprimé. Le frontend masque le bouton Supprimer dans ce cas.
Validation planningÀ la création/modification d'un créneau, le backend vérifie que la matière est dans matiere_ids du niveau de la classe. Retourne 400 si refusé.

Gestion de l'ordre

L'algorithme recompact(targetId, targetOrdre) dans NiveauxService :

  1. Récupère tous les niveaux triés par ordre
  2. Retire targetId de la liste
  3. L'insère à la position targetOrdre (clampé entre 0 et la longueur de la liste)
  4. Réassigne les indices 0, 1, 2… à chaque élément dans l'ordre obtenu

Ce même algorithme est appelé après chaque create, update et delete, garantissant que les ordres sont toujours consécutifs et sans trous.

Comportement de la page /niveaux

  • Chaque niveau s'affiche en carte avec : nom, nombre de classes (badge), ordre, description, matières autorisées (badges)
  • Modifier : ouvre une modale avec les champs nom, ordre, description et le sélecteur de matières
  • Supprimer : disponible uniquement si count === 0. Demande confirmation avant suppression.
  • Les niveaux orphelins (id: null) peuvent être modifiés (→ crée le niveau en base) mais pas supprimés via l'API (le bouton retire l'entrée de la liste locale uniquement)
  • Carte « + Nouveau niveau » à la fin de la grille, lien vers /niveaux/nouveau
