Architecture globale

Frontend (React) │ ├─ Pages → hooks dédiés → 1 appel API / page │ ↘ GET /read/* │ ├─ Contexts write-only → POST/PATCH/DELETE │ ↘ Socket.IO notifyDataChange() │ └─ ViewingContext → mode archive (snapshot) Backend (NestJS) │ ├─ Write Controllers → collections write (source de vérité) ├─ Read Controller → collections read dénormalisées ├─ ViewBuilder → write → read (bulkWrite / upsert) ├─ EventsGateway → Socket.IO └─ ApiLoggerMiddleware→ logs structurés

Flux de données

  1. Lecture : chaque page appelle uniquement l'endpoint /read/* nécessaire à son affichage. Aucun préchargement global.
  2. Écriture : les formulaires appellent les endpoints write (POST/PATCH/DELETE) sur les contrôleurs métier.
  3. Projection : après chaque écriture, le ViewBuilderService reconstruit les documents des collections read impactées. Les Change Streams sont utilisés si disponibles ; sinon les write controllers déclenchent explicitement la projection.
  4. Temps réel : le backend émet un événement Socket.IO. Le frontend reçoit l'événement, appelle notifyDataChange(), puis la page re-fetch son endpoint read après un petit délai.
  5. Archive : en consultation d'une année terminée, le frontend lit un snapshot et passe en lecture seule.
