Flux de données

  1. Chargement initial : chaque contexte React fait un fetch() vers l'API au montage. Si l'API est indisponible, le fallback mock est utilisé.
  2. Mutations : les formulaires appellent l'API via POST/PATCH/DELETE. Le contrôleur NestJS exécute l'opération puis émet un événement Socket.IO.
  3. Temps réel : le socketService client écoute les événements et met à jour les contextes React automatiquement.
  4. Mode archive : quand l'utilisateur consulte une année terminée, le ViewingContext charge un snapshot complet depuis GET /annees/:id/snapshot et toutes les pages affichent les données du snapshot en lecture seule.
