Emplois du temps : créneaux hebdomadaires par classe

Schéma Creneau

{
  classe_id:       String  (requis)  // référence Classe
  matiere_id:      String  (requis)  // référence Matiere
  matiere_nom:     String  (requis)  // dénormalisé pour l'affichage
  matiere_couleur: String  (défaut "#2563eb")
  jour:            Enum    ['Lundi'..'Samedi']
  heure_debut:     String  (requis)  // "08:00"
  heure_fin:       String  (requis)  // "09:00"
  salle:           String  (requis)  // "Salle 101"
  enseignant:      String  (défaut "")
}

Endpoints — Write

MéthodeRouteDescription
POST/planningCréer un créneau
PATCH/planning/:idModifier un créneau
DELETE/planning/:idSupprimer un créneau
POST/planning/merge/:classeIdFusionner les créneaux adjacents identiques d'une classe

Endpoints — Read

MéthodeRouteDescription
GET/read/planning/classesToutes les classes avec leur nombre de créneaux
GET/read/planning/classe/:idClasse + ses créneaux + ses matières autorisées

Validation — Matière par niveau

À chaque création ou modification de créneau, le backend vérifie que la matière est autorisée pour le niveau de la classe :

  • Si le niveau de la classe a des matiere_ids configurés → la matière doit figurer dans cette liste
  • Si matiere_ids est vide → toutes les matières sont autorisées
  • En cas de refus : 400 Bad Request — « Cette matière n'est pas autorisée pour le niveau X »

Grille horaire

Plage : 07:00 → 18:00 par pas de 30 minutes.
Jours : Lundi, Mardi, Mercredi, Jeudi, Vendredi, Samedi.

Dénormalisation

matiere_nom et matiere_couleur sont stockés directement dans le créneau. Cela permet d'afficher le calendrier sans jointure.
