📝 Saisie des Notes

Page /notes

Source des données

La page charge sa configuration initiale via GET /read/notes (hook useNotesPageData, channel socket 'notes') :

  • Toutes les matières (matieres[])
  • Tous les élèves (eleves[])
  • Toutes les notes existantes (notes[])

Les élèves et notes sont ensuite filtrés côté frontend selon la classe et la matière sélectionnées.

Composants

ComposantRôle
NotesFiltersBarre de sélection structurée : niveau → classe (popover) + matière (autocomplete) + trimestre
NotesStatsBarBande : « X / Y saisies » + moyenne de classe
NotesTableTableau de saisie avec inputs stylés par valeur et colonne statut

NotesFilters — Navigation structurée

Le flux de sélection suit le parcours : niveau → classe → matière → trimestre → Charger.

Sélection de classe — popover niveau + modal classes

Même pattern que ElevesFiltersBar :

  • Un bouton Niveau / Classe ouvre un Popover listant les niveaux (GET /read/niveaux, chargé une seule fois via useRef)
  • La sélection d'un niveau ferme le Popover et ouvre un modal overlay listant les classes de ce niveau (GET /read/niveaux/:niveau/classes)
  • La sélection d'une classe ferme le modal et affiche le label Niveau · Nom classe sur le bouton
  • Changer de classe réinitialise les lignes de saisie

Sélection de matière — autocomplete

  • Composant SearchInputSuggestions avec filtre client-side sur les matières déjà chargées
  • Recherche sur le nom et le code de la matière, debounce 150ms
  • Chaque suggestion affiche le nom + le code (sublabel)
  • Changer de matière réinitialise les lignes de saisie

Date d'évaluation

Le champ date n'est plus exposé dans l'interface. La date est générée automatiquement à today (new Date().toISOString().split('T')[0]) au moment de l'enregistrement.

NotesTable — Saisie en lot

Affiche une ligne par élève de la classe sélectionnée :

  • note-field : input numérique, min 0, max 20, step 0.5. La valeur est clampée à [0, 20] dès la saisie. Couleur dynamique : vert ≥ 10, rouge < 10
  • comment-field : champ texte libre
  • Statut : badge Existante (update), Nouvelle (create) ou — (pas de note)

Enregistrement

  • Validation côté frontend avant envoi : toute valeur hors de [0, 20] bloque le save avec un message d'erreur
  • Pour chaque ligne avec une note : POST /notes (création) ou PATCH /notes/:id (mise à jour)
  • Après le save, les lignes sont mis à jour localement — les valeurs saisies restent affichées sans rechargement
  • Les notes nouvellement créées passent au statut Existante immédiatement
  • Le re-fetch complet depuis l'API se fait en arrière-plan via le socket (channel: 'notes')

Lecture seule archive

En mode archive (ViewingContext.isViewingArchive) :

  • Tous les champs sont disabled
  • Le bouton d'enregistrement est masqué
  • Une alerte orange indique le mode consultation

Sous-titre dynamique

Le PageHeader affiche : NomClasse · NomMatière — Trimestre X une fois la classe et la matière sélectionnées, ou le texte par défaut Sélectionnez niveau, classe et matière.
