📚 Module Matières

Matières avec coefficients par niveau scolaire, couleurs et protection à la suppression

Schéma MongoDB

{
  nom:          String  (requis)           // "Mathématiques"
  code:         String  (requis)           // "MATH" (max 6 car.)
  coefficient:  Number  (défaut 1)         // héritage — ignoré si coefficients[] présent
  coefficients: [{ niveau: String, coefficient: Number }]  // par niveau scolaire
  description:  String  (optionnel)
  couleur:      String  (optionnel)        // "#2563eb"
}

Coefficients par niveau : chaque matière peut définir un coefficient différent selon le niveau scolaire. Exemple : Mathématiques → 6ème × 4, Terminale × 5. Le champ coefficient global est conservé pour rétrocompatibilité.

Endpoint Read

RouteParamsRetour
GET /read/matierespage, limit, niveau (optionnel){ items, total, page, limit, totalPages }

Le paramètre niveau filtre via coefficients.niveau en MongoDB — retourne uniquement les matières définissant un coefficient pour ce niveau.

Règles métier

  • Chaque entrée de coefficients[] doit avoir un niveau non vide et un coefficient > 0
  • Un même niveau ne peut apparaître qu'une seule fois dans coefficients[]
  • Protection suppression : si la matière est référencée dans au moins une note (notes) ou un créneau (creneaux), le DELETE retourne 400 Bad Request avec un message indiquant le nombre de dépendances

Résolution du coefficient dans le bulletin

Le ReadService.getBulletin() résout le coefficient de la matière pour l'élève via la logique suivante :

const niveau = classe.niveau;  // niveau de la classe de l'élève
const found = matiere.coefficients.find(c => c.niveau === niveau);
const coeff = found ? found.coefficient
            : coefficients.length === 1 ? coefficients[0].coefficient
            : matiere.coefficient ?? 1;  // fallback héritage
