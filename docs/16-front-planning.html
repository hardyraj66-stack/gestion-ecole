Sidebar — Navigation par niveaux

Les niveaux sont chargés depuis GET /read/niveaux et triés selon leur ordre. Chaque niveau est un accordéon (popover) qui liste les classes qui lui appartiennent. Cliquer sur une classe la sélectionne et charge son planning.

  • Badge sur chaque niveau : nombre de classes
  • Badge sur chaque classe : nombre de créneaux (_creneauxCount)
  • Encart « Sélection » en bas de sidebar : nom, niveau, salle de la classe active

Barre d'outils (mode édition uniquement)

  • Annuler (Ctrl+Z) — revient en arrière sur la dernière action
  • Rétablir (Ctrl+Y / Ctrl+Shift+Z) — refait l'action annulée
  • Pile undo/redo illimitée par session
  • Hint contextuel : Clic → créer · Glisser → déplacer · Clic droit → options

Grille horaire

Plage 07:00 → 18:00 · pas de 30 minutes · colonnes : Lundi → Samedi.

  • Chaque créneau est un bloc coloré (couleur de la matière) affichant : nom matière, horaires, salle, enseignant
  • Les créneaux s'étirent sur plusieurs lignes selon leur durée (CSS grid row-span)

Modale de création / édition

  • Matière — liste filtrée par les matières autorisées du niveau (via config du niveau)
  • Jour, heure début, heure fin
  • Salle :
    • Mode fixe : champ en lecture seule, salle imposée par la classe
    • Mode variable : sélecteur avec vérification de disponibilité en temps réel — légende verte/rouge selon conflit
  • Enseignant (optionnel)

Filtre matières par niveau

Les options de la liste matières sont filtrées en fonction de la configuration du niveau de la classe sélectionnée :

  • Si le niveau a des matiere_ids configurés → seules ces matières apparaissent dans le select
  • Si matiere_ids est vide → toutes les matières sont proposées

Le backend applique également cette règle de validation à la création/modification d'un créneau et renvoie 400 Bad Request si la matière n'est pas autorisée.

État local et undo/redo

L'état du planning est géré par le hook usePlanningState. Chaque action (créer, modifier, déplacer, supprimer, dupliquer) est empilée dans undoStack. Annuler une action la bascule dans redoStack.
