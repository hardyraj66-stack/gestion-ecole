📝 Module Notes & Bulletin

Notes par élève/matière/trimestre, calcul du bulletin et des moyennes

Schéma Note

{
  eleve_id:    String  (requis) // référence Eleve._id
  matiere_id:  String  (requis) // référence Matiere._id
  valeur:      Number  (requis) // 0 à 20, step 0.5
  trimestre:   Enum    [1, 2, 3]
  date:        String  (requis) // "2024-10-15" — date de saisie (today au moment de l'enregistrement)
  commentaire: String  (optionnel)
}

Validation

  • valeur doit être comprise entre 0 et 20 inclus
  • Le backend (NotesService) rejette avec 400 Bad Request toute valeur hors de cette plage
  • Le frontend clamp la valeur à [0, 20] dès la saisie, et re-valide avant l'envoi

Endpoint Read — Bulletin

GET /read/bulletin/:eleveId?trimestre=1 retourne un objet complet :

{
  eleve: { ... },
  classe: { ... },
  bulletin: [{
    matiere_id:  "...",
    matiere_nom: "Mathématiques",
    code:        "MATH",
    coefficient: 4,          // résolu selon le niveau de la classe de l'élève
    notes:       [15, 12.5, 14],
    moyenne:     13.8        // moyenne arithmétique arrondie 1 décimale
  }]
}

Résolution du coefficient : le ReadService cherche dans matiere.coefficients[] l'entrée dont le niveau correspond à la classe de l'élève. Si non trouvé, il utilise le coefficient global héritage (matiere.coefficient).

Calcul de la moyenne générale (frontend)

totalCoef = Σ coefficient (matières avec notes)
somme     = Σ (moyenne_matière × coefficient)
générale  = Math.round((somme / totalCoef) * 10) / 10
