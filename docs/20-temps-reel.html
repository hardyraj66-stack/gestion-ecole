Connexion authentifiée. La gateway EventsGateway.handleConnection exige un jeton JWT valide dans le handshake, revérifie le compte en base (actif / non archivé) et déconnecte toute connexion non autorisée. Le temps réel n'est donc accessible qu'aux clients authentifiés. Voir Authentification & Comptes.

Écoute dans les contextes

useEffect(() => {
  const unsub1 = onEvent<Classe>('classe:created', (data) => {
    setClasses(prev => {
      if (prev.find(c => c.id === data.id)) return prev; // anti-doublon
      return [...prev, data];
    });
  });

  const unsub2 = onEvent<Classe>('classe:updated', (data) => {
    setClasses(prev => prev.map(c => c.id === data.id ? data : c));
  });

  const unsub3 = onEvent<{id:string}>('classe:deleted', (data) => {
    setClasses(prev => prev.filter(c => c.id !== data.id));
  });

  return () => { unsub1(); unsub2(); unsub3(); };
}, []);

Cas spécial Salles : les opérations CRUD ne mettent pas à jour le state local après le POST/PATCH/DELETE. Seul l'événement socket fait la mise à jour, pour éviter les doublons.
