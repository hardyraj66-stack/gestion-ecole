🏫 Gestion des Classes

Pages /classes, /classes/nouvelle, /classes/:id/eleves

Liste des classes — /classes

La page utilise GET /read/classes?page=X&limit=8&search=.... La pagination est 100% côté serveur. Le frontend ne fait plus de paginate() local.

  • Réponse API : { items, total, page, limit, totalPages }
  • Recherche : filtre MongoDB via $regex sur nom
  • Carte « + » : affichée seulement sur la dernière page et hors mode archive

ClasseCard — contenu

  • Header : nom de la classe + badge niveau
  • Meta : salle, année scolaire
  • Jauge : « Élèves X / Y » + barre de progression
  • Actions : Élèves (lien), Planning (lien), ✕ Supprimer (masqué en mode archive)

Créer une classe — /classes/nouvelle

Formulaire avec : nom, niveau, année scolaire, capacité, mode salle, salle. Les salles sont chargées à la demande via GET /read/create-classe.

Élèves d'une classe — /classes/:id/eleves

La page appelle GET /read/classes/:id/eleves?page=X&limit=10&search=....

  • ClasseInfoBar : compteurs + champ recherche
  • ElevesTable : tableau déjà paginé par le backend
  • Recherche par nom/prénom faite côté MongoDB
