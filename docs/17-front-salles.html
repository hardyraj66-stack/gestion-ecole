🏠 Gestion des Salles

Page /salles

Lecture des salles

La liste appelle GET /read/salles?page=X&limit=8. La pagination est entièrement côté serveur.

  • Réponse : { items, total, page, limit, totalPages }
  • Le frontend affiche items seulement

Mutation des salles

  • POST /salles, PATCH /salles/:id, DELETE /salles/:id
  • Après chaque mutation : émission socket + ViewBuilder.onSalleWrite()
  • Le frontend re-fetch automatiquement la liste via notifyDataChange()

Disponibilité

L’endpoint GET /salles/disponibles reste côté write, car il sert le formulaire de création de créneau et dépend de contraintes métier en temps réel.
