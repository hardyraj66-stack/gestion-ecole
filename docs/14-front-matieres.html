📚 Gestion des Matières

Pages /matieres, /matieres/nouvelle

Liste — /matieres

La page appelle GET /read/matieres?page=X&limit=8&niveau=.... La pagination est côté serveur.

  • Réponse : { items, total, page, limit, totalPages }
  • La carte « + Nouvelle matière » n'apparaît que sur la dernière page
  • Un Select de niveau dans le PageHeader filtre les matières : seules celles ayant un coefficient défini pour ce niveau sont retournées
  • Changer le niveau remet la pagination à 1

Composant MatiereCard

Chaque carte affiche le code coloré, le nom, la description et les coefficients par niveau sous forme de badges.

Mode affichage

  • Badges niveau × coeff pour chaque entrée de coefficients[]
  • Si coefficients[] est vide, affiche le coefficient global héritage (matiere.coefficient)
  • Bouton crayon pour passer en mode édition (masqué en lecture seule archive)
  • Bouton Supprimer avec useConfirm() — si le backend renvoie une erreur 400, un second dialog affiche le message de blocage

Mode édition inline

Le clic sur le crayon bascule la carte en mode formulaire sans navigation :

  • Champs éditables : nom (requis), description, coefficients par niveau
  • Tableau de coefficients : chaque ligne affiche le niveau + un input numérique (min 0.5, max 20, step 0.5) + bouton ✕ pour retirer le niveau
  • Un select « + Ajouter un niveau » liste les niveaux non encore assignés (chargés via GET /read/niveaux au montage)
  • Enregistrement via PATCH /matieres/:id avec { nom, description, coefficients[] }
  • Mise à jour locale optimiste via le callback onUpdated(updatedMatiere) — pas de rechargement complet
  • Annuler restaure les valeurs initiales

Création — /matieres/nouvelle

Layout 2 colonnes : formulaire à gauche (max 900px) + aperçu temps réel à droite (300px fixe).

En mode archive, redirection automatique vers la liste.

Formulaire

  • Nom et Code (obligatoires, code forcé en majuscules, max 6 caractères)
  • Description (optionnelle)
  • Couleur via ColorPicker (8 couleurs prédéfinies)
  • Coefficients par niveau : pré-remplis avec tous les niveaux (coefficient 1 par défaut), possibilité de retirer ou d'ajouter des niveaux

Validation

  • Au moins un niveau avec coefficient défini
  • Tous les coefficients doivent être > 0

Aperçu — MatierePreview

Reflète en temps réel : bloc code coloré, nom, description, badges coefficients par niveau.

Mises à jour temps réel

Le hook useMatieresListData(page, niveau) est abonné au channel 'matieres'. Toute mutation (création, édition, suppression) déclenche un re-fetch silencieux depuis le socket.

L'édition inline et la suppression utilisent également un état local localItems pour une mise à jour optimiste instantanée avant que le socket ne confirme.
