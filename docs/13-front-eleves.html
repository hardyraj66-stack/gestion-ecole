👥 Gestion des Élèves

Pages /eleves, /eleves/nouveau, /eleves/:id, /eleves/:id/bulletin

Liste — /eleves

La page appelle GET /read/eleves?page=X&limit=12&search=...&classeId=....

  • Pagination backend : le frontend ne stocke que la page courante
  • Recherche backend : MongoDB filtre via $or sur nom et prenom
  • Filtre classe backend : MongoDB filtre sur classe_id
  • Réponse API : { eleves, classes, total, totalAll, page, limit, totalPages }

Important : les données proviennent de la collection read_eleves, qui contient déjà classe_nom et classe_niveau.

Barre de filtres — ElevesFiltersBar

La barre de filtres propose une navigation progressive en deux étapes : niveau → classe.

Recherche avec auto-complétion — SearchInputSuggestions

Composant réutilisable (src/components/shared/SearchInputSuggestions.tsx) qui remplace le SearchInput basique.

  • Debounce de 250 ms avant chaque appel de suggestions
  • Suggestions à partir de 2 caractères saisis, via fetchSuggestions(query) passé en prop
  • Sur la page élèves, appelle GET /read/eleves?page=1&limit=8&search=... pour produire les suggestions
  • Chaque suggestion affiche le nom complet + la classe de l'élève (sublabel)
  • Navigation clavier : ↑ / ↓ pour parcourir, Entrée pour sélectionner, Échap pour fermer
  • Bouton ✕ pour effacer la saisie

Filtre niveau → popover niveaux

  • Un bouton Niveau / Classe ouvre un Popover (src/components/shared/Popover.tsx) listant les niveaux disponibles
  • Les niveaux sont chargés via GET /read/niveaux une seule fois (mémoïsé via useRef)
  • Chaque entrée affiche le niveau + le nombre de classes (Badge)

Sélection de classe → modal classes

  • La sélection d'un niveau ferme le popover et ouvre un modal overlay listant les classes de ce niveau
  • Les classes sont chargées via GET /read/niveaux/:niveau/classes à chaque sélection de niveau
  • Chaque classe affiche : nom, badge Complète si pleine, compteur élèves/capacité, barre de progression
  • Un bouton ← Niveaux permet de revenir au popover précédent
  • La sélection d'une classe déclenche l'appel GET /read/eleves?classeId=... et ferme le modal

Réinitialisation

Un bouton Réinitialiser apparaît dès qu'un filtre ou une recherche est actif. Il vide tous les états (search, classeId, niveau) et remet la pagination à 1.

Fiche élève — /eleves/:id

Page de détail chargée via GET /read/eleves/:id/fiche. Réponse : { eleve, classe, salleActuelle, creneaux, anneeActive }.

La page est en lecture seule (readOnly) lorsque l'utilisateur consulte une archive via ViewingContext.

Mise en page — grille 2 colonnes

La page utilise une grille CSS (.fiche-layout) à 2 colonnes : colonne gauche fixe à 340 px, colonne droite flexible.

ColonneComposants (ordre)
GaucheFicheIdentite → FicheFamille
DroiteFicheShortcuts → FicheAvertissements → FicheAssiduité

Composants de la fiche

ComposantDescription
FicheIdentiteAvatar, nom, genre, date de naissance + âge, classe, salle actuelle. Pour les classes à salle variable, calcule la salle du créneau en cours ou du prochain créneau du jour.
FicheFamillePère, mère (formulaire inline avec statut vivant/décédé), tuteur légal. Enregistrement via PATCH /eleves/:id (contexte useEleves).
FicheShortcuts4 raccourcis en grille 2×2 (Notes, Bulletin, Classe, Emploi du temps). Placé en haut de la colonne droite. Chaque carte a bordure, fond grisé et hover bleu.
FicheAvertissementsListe + formulaire d'ajout (type, date, année scolaire, motif, commentaire). Appels directs à /suivi/:eleveId/avertissements.
FicheAssiduitéDeux sections (absences, retards), chacune avec liste + formulaire. Appels directs à /suivi/:eleveId/absences et /suivi/:eleveId/retards.

Module Suivi — endpoints

Le module SuiviModule (NestJS) gère avertissements et absences/retards indépendamment du flux CQRS.

MéthodeRouteDescription
GET/suivi/:eleveId/avertissementsListe des avertissements
POST/suivi/:eleveId/avertissementsCréer un avertissement
DELETE/suivi/avertissements/:idSupprimer un avertissement
GET/suivi/:eleveId/absencesListe des absences
POST/suivi/:eleveId/absencesCréer une absence
GET/suivi/:eleveId/retardsListe des retards
POST/suivi/:eleveId/retardsCréer un retard
DELETE/suivi/absences/:idSupprimer absence ou retard

Données famille dans le schéma Eleve

Les champs pere, mere, tuteur sont stockés dans le schéma Eleve (MongoDB) et répliqués dans read_eleves via ViewBuilder.rebuildEleves().

  • pere / mere : { nom, prenom, telephone, email, statut: 'vivant' | 'decede' }
  • tuteur : { nom, prenom, telephone, email, lien }

Inscription — /eleves/nouveau

Le formulaire charge les niveaux disponibles via GET /read/create-eleve (alias de GET /read/niveaux) au montage, une seule fois. En mode archive, la page redirige automatiquement.

La sélection d'une classe suit le même parcours niveau → popover classes que dans la liste, via GET /read/niveaux/:niveau/classes.

Bulletin — /eleves/:id/bulletin

Le bulletin est chargé via GET /read/bulletin/:eleveId?trimestre=X. Chaque changement d'onglet trimestre déclenche un nouvel appel API ciblé.

  • Données renvoyées : { eleve, classe, bulletin[] }
  • Le frontend calcule la moyenne générale pondérée à partir de bulletin[]
